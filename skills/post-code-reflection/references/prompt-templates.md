# 代码后反思提示词模板

## 需求前最小设计

```txt
我要实现什么：
输入是什么：
输出是什么：
核心流程：
可能的边界情况：
```

## 让 AI 实现

```txt
根据下面设计实现代码。
要求：
- 保持结构清晰
- 不要过度封装
- 给出关键设计理由
- 标出你假设的前提

设计：
{你的设计}
```

## 代码生成后自审

```txt
请严格 review 你刚才生成的代码：
- 可能的 bug
- 边界情况
- 可维护性问题
- 可以简化的地方
- 哪些地方未来扩展会痛苦
```

## 解释与对比

```txt
请解释这段代码：
- 为什么这样设计？
- 关键逻辑是什么？
- 有哪些隐含假设？
- 有没有更简单或更稳妥的方案？
- 如果数据规模变大，当前实现会在哪里出问题？
```

## 基于审查重构

```txt
基于 review 结果重构代码。
要求：
- 不改变原有功能
- 结构更清晰
- 删除不必要复杂度
- 保留必要的边界判断
- 说明每个修改点为什么值得做
```

## 反向练习

```txt
基于这段代码，给我 3 个改进题。
要求：
- 每题都能在 15-30 分钟内手写完成
- 覆盖可读性、边界处理、测试或性能中的不同维度
- 不直接给答案，只给验收标准
```

## 经验沉淀

```txt
把这次实现总结成一条可复用开发经验：
- 适用场景
- 关键原则
- 常见坑
- 下次遇到类似问题怎么做
```
